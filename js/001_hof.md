# 高阶函数

在JavaScript中，函数是一等公民。高阶函数，简单理解就是返回函数的函数，其实在项目开发中已经用到了很多，在这里略作整理。

## 闭包

返回的函数可能会使用函数内部的环境，这时候就会产生闭包，例如：

```javascript
function inc(initial) {
  let counter = initial;
  return function() {
    counter++;
    return counter;
  };
}
```

在旧版本的JS中，经常使用`var`关键词来声明循环变量，当结合闭包使用时，容易产生问题：

```javascript
for (var i = 0; i < 5; i++) {
  setTimeout(() => {
    console.log(i); // 永远打印5
  }, 500);
}
```

这里将会永远打印5, 因为闭包捕获的是变量i的引用，而不是值。

有两种解决方案：1. 立即执行函数; 2. 关键词`let`。

第一种方案是通过立即执行函数创建一个新的环境，让新的环境立即获得变量的值，再让闭包引用新环境里的值：

```javascript
for (var i = 0; i < 5; i++) {
  (function(index){setTimeout(() => {
    console.log(index); // 正常打印
  }, 500);})(i)
}
```

第二种方案则是使用新的关键词`let`，JS引擎会在每次执行时创建一个新的变量并赋予新的值，因此闭包捕获到的永远是新的值的引用。

```javascript
for (let i = 0; i < 5; i++) {
  setTimeout(() => {
    console.log(i); // 正常打印
  }, 500);
}
```

## 函数作为参数

函数能作为返回值，自然也能作为参数，比如下面的`once`函数，它接受一个函数作为参数，然后返回另一个函数：

```javascript
function once(f) {
  let fn = f;
  let result = null;
  return function(...args) {
    if (fn != null) {
      result = fn.apply(this, args);
      fn = null; // 解除引用，释放内存
    }
    return result;
  }
}
```

### 节流和防抖

节流和防抖是‘函数作为参数’的两个应用场景，其中节流指指定周期内只执行1词，防抖则是降低时间发生的频率，分别有如下实现：

```javascript
// 节流
// 每delay ms只执行一次
function throttle(func, delay) {
  let lastCalled = 0;
  return function(...args) {
    let now = Date.now();
    if (now - lastCalled > delay) {
      lastCalled = now;
      func.apply(this, args);
    }
  };
}

// 防抖
function debounce(func, delay) {
  let timer = null;
  return function(...args) {
    const context = this;
    if (timer != null) {
      clearTimeout(timer);
    }
    timer = setTimeout(() => {
      func.apply(this, args);
      timer = null;
    }, delay);
  };
}
```

节流和防抖非常相似，可以用两个比喻进行区分：

- **节流**：通过按键释放技能，在一个冷却时间内，不管按多少次按键，都只能释放一次；
- **防抖**：电梯门打开后就触发“关门函数”，但是每次有人点开门都会延迟“关门函数”的执行；

## 柯里化

柯里化是函数式变成中的概念，它把一个复杂的函数改写为数个单参数函数，并在最后一个参数集齐时才真正求值。

例如函数

```JavaScript
function add(a, b) {
  return a + b;
}
```

可以柯里化为

```JavaScript
function add(a) {
  return funcion(b) {
    return a + b;
  };
}
```

柯里化带来的好处包括：

1. 可复用性：修改参数就可以复用逻辑，无须重新编写
2. 延迟求值：集齐参数才会求值
3. 可读性：分阶段求值可读性高于一次性全写完
